## 1. Entity Facet 为空

**现象**：
- 知识库统计 `entity: 0`
- `~/linglong/wiki/entity/` 目录为空

**调查过程**：
1. 检查原始 wiki `~/.openclaw/workspace/memory/wiki/entity/` — 目录不存在
2. `grep -r "^type: entity"` — 0 条结果
3. 检查同步适配器 `TYPE_TO_FACET` 映射表 — `"entity": EntityFacet.ENTITY` 已配置
4. 确认默认分支 — 未映射类型默认走 `SOURCE`

**关键数据**：
原始 wiki 类型分布：project(32), source-code-note(29), experience(15), concept(11), methodology(6)...

**结论**：原始 wiki 中本来就没有 entity 类型的文件，不是同步 bug。

**关联**：`src/linglong/knowledge/sync/openclaw.py:64-103`

## 2. 34 个文件数据库无记录

**现象**：`linglong lint` 报告 `index_consistency` 34 个：
- `concept/b3356305-wiki-目录结构设计.md`

**结果**：index_consistency 归零。

## 3. LightSleep/Deep/REM 清理

**现象**：`content_conflict` 报告 216 个 `light sleep` 标题重复，另有 `Deep Sleep` 25 个、`REM Sleep` 24 个。

**原因**：OpenClaw memory 内部 Sleep 机制产生的系统日志，无知识价值。

**解决**：
1. 删除文件：Light Sleep × 25、Deep Sleep × 25、REM Sleep × 25
2. 清理数据库记录：57 条
3. 修改 `openclaw.py`，sync 时自动过滤 Sleep 日志

**结果**：
- content_conflict：216 → 144
- 总条目：457 → 375

**关联**：`src/linglong/knowledge/sync/openclaw.py`

## 4. lint content_conflict 误报修复

**现象**：清理 Sleep 后仍有 140 个 content_conflict，调查发现大部分是 diary 和 task-record 的标题重复（如 `2026-04-23`、`她的日记`）。

**原因**：lint 规则对所有文件检测标题重复，但 memory 同步的日记（日期标题）和任务记录（同一任务多次记录）标题重复是正常行为。

**解决**：修改 `lint.py`，对 `_subdir` 为 `diary` 或 `task-record` 的文件豁免标题重复检测。

**结果**：content_conflict 140 → 1

**关联**：`src/linglong/knowledge/lint.py`
