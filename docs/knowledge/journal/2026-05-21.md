## 2. 孤儿资源检测与收窄

**实现**：`lint.py` 新增 `check_orphans()` 方法：
- 遍历所有实体，解析 `[[target]]` 引用
- 统计每个实体被引用次数
- 从未被引用的标记为孤儿

**问题**：首次运行发现 336 个孤儿（92.5%），因为知识库以独立文档为主（experience/source），几乎不使用 `[[链接]]` 语法。

**收窄**：将检测范围限制为 `concept` 和 `entity` facet（知识网络节点），其余 facet 是独立文档不需要被引用。

**增强**：检测到孤儿后，扫描其他文档是否提到该标题，输出引用建议：
```
[orphan] 0912359d... 孤儿资源：未被任何页面引用，20 篇文档提到此标题
        → 建议在 1430b1c8... (linglong-pipeline) 中添加 [[链接]]
```

**结果**：孤儿从 336 降至 14 个，都是有价值的引用建议。

**关联**：`src/linglong/knowledge/lint.py`、`src/linglong/cli.py`

## 3. 向量语义去重

**实现**：增强 `check_content_conflicts()`：
- 遍历每个实体，调用 `search_similar`（内容截断 500 字符）

## 4. 定期巡检调度

**实现**：新增 `lint_schedule.py` 模块：
- `parse_cron_to_schedule()` — 解析 `分 时 * * *` 格式的 cron 表达式
- `run_lint_and_log()` — 执行一次巡检并追加写入 `~/.linglong/logs/lint-schedule.log`
- `run_daemon()` — 守护进程模式，先执行一次全量巡检，再按 schedule 循环等待，支持 SIGTERM/SIGINT 优雅退出

**CLI 入口**：
- `linglong lint --daemon` — 后台守护进程模式
- `linglong lint --run-scheduled` — 单次执行（用于系统 cron）

**配置**：`init.py` 交互式初始化新增定时巡检问题，`.linglong.yaml` 新增 `lint_schedule` 字段。

**关联**：`src/linglong/knowledge/lint_schedule.py`、`src/linglong/cli.py`、`src/linglong/core/config.py`

## 8. OpenClaw Phase 0.1 memory_search 验证

| 方案 | 操作 | 结果 | 原因 |

**当前状态**：
