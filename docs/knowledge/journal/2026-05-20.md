## 1. 文件名 ID 前缀影响可读性

**背景**：用户强调"美观整齐度很重要"，当前文件名格式 `{id[:8]}-{slug}.md`（如 `cc039c6d-技能触发失败排查.md`）中 ID 前缀占用空间且降低人类可读性。

**方案对比**：

| 方案 | 示例 | 优点 | 缺点 |
|------|------|------|------|
| A. 纯 slug | `技能触发失败排查.md` | 最好看 | 反查需遍历/改 schema，碰撞风险 |
| B. slug + ID 后缀（选中） | `技能触发失败排查-cc039c6d.md` | slug 打头，O(1) 反查 | 尾部有 ID |
| C. 保持现状 | `cc039c6d-技能触发失败排查.md` | 无需改动 | ID 前缀最丑 |

**决策**：选择方案 B。slug 放前保证了目录列表一眼识别内容，ID 后缀保证唯一性和 O(1) 反查，实现成本最低。

**实现**：
1. `store.py`：`_get_entity_path()` 文件名拼接顺序调整
2. `lint.py`：`check_index_consistency` ID 提取改为 `rsplit("-", 1)[-1]`
3. 批量重命名 331 个现有文件

**结果**：253 个测试全部通过。

**关联**：BACKLOG-004（状态更新为已完成）

## 2. Claude Code 接入 wiki 查询

**需求**：用户提出"接入到 claude code，使 claude code 能实现 wiki 查询"。

**调研**：参考 Mem0 的成熟方案，其通过 MCP Server 暴露 `search_memories` / `add_memory` 工具，让 Claude Code 自主决定何时查/写。

**方案**：为 Linglong 创建 MCP Server

**实现**：
1. 新增 `src/linglong/mcp/` 模块（tools.py + server.py + __init__.py）
2. 暴露 5 个工具：
   - `search_wiki` — FTS5 全文搜索
   - `search_similar` — 向量语义搜索（失败回退 FTS5）
   - `read_entity` — 读取完整内容
   - `write_entity` — 写入新知识
   - `list_entities` — 浏览最近条目
3. 安装 `mcp` 依赖，更新 `pyproject.toml`
4. 11 个测试覆盖全部工具 + 错误场景
5. 264 个全量测试通过

**配置**：
```json
// ~/.claude/mcp.json
{
  "mcpServers": {
    "linglong": {
      "command": "bash",
      "args": ["-c", "cd /home/user/projects/linglong && source venv/bin/activate && python -m linglong.mcp"]
    }
  }
}
```

**状态**：已完成
