## 1. 文件名 ID 前缀影响可读性

**背景**：用户强调"美观整齐度很重要"，当前文件名格式 `{id[:8]}-{slug}.md`（如 `cc039c6d-技能触发失败排查.md`）中 ID 前缀占用空间且降低人类可读性。

**方案对比**：

| 方案 | 示例 | 优点 | 缺点 |
|------|------|------|------|
| A. 纯 slug | `技能触发失败排查.md` | 最好看 | 反查需遍历/改 schema，碰撞风险 |
| B. slug + ID 后缀（选中） | `技能触发失败排查-cc039c6d.md` | slug 打头，O(1) 反查 | 尾部有 ID |
| C. 保持现状 | `cc039c6d-技能触发失败排查.md` | 无需改动 | ID 前缀最丑 |

**决策**：选择方案 B。slug 放前保证了目录列表一眼识别内容，ID 后缀保证唯一性和 O(1) 反查，实现成本最低。

**实现**：
1. `store.py`：`_get_entity_path()` 文件名拼接顺序调整
2. `lint.py`：`check_index_consistency` ID 提取改为 `rsplit("-", 1)[-1]`
3. 批量重命名 331 个现有文件

**结果**：253 个测试全部通过。

**关联**：BACKLOG-004（状态更新为已完成）

## 3. MCP 搜索返回量优化

**背景**：Claude Code 调用 `search_and_read` 搜索 "MCP" 时，一次返回 5 篇长文共 120KB，Token 消耗过大，Claude Code 被迫把内容写入本地文件再读取。

**方案**：`search_and_read` 增加 `max_content_length` 参数（默认 2000 字符），超长自动截断并标注 `... [truncated]`，返回 `truncated: true` 提示 Agent 按需调 `read_entity` 读全文。

**验证**：搜索 "MCP" 从 120KB 降至约 10KB，Agent 可一次性消化。

**关联**：`src/linglong/mcp/tools.py`

## 6. 模板体系

**背景**：Agent 通过 `write_entity` 写入知识库时，格式和结构不一致。需要为每个 facet 提供标准模板，让 Agent 写入时有结构参考。

**设计**：
- 参考 OpenClaw 的 `templates/concept.md`、`experience.md`、`project.md`
- 适配 Linglong 的 7 分面分类，新增 methodology/source/entity/synthesis/personal
- 模板存放于 `~/linglong/templates/`，含 YAML frontmatter + Markdown 占位符

**模板清单**：

| 模板 | facet | 用途 |
|------|-------|------|
| concept.md | concept | 概念定义、核心要点、实际应用 |
| experience.md | experience | 背景、问题、解决方案、踩坑记录 |
| project.md | experience | 项目目标、进度、关键决策、学习笔记 |
| methodology.md | methodology | 流程、检查清单、常见误区 |
| source.md | source | 外部资料、核心观点、摘录、待验证 |
| entity.md | entity | 专有名词、关键特性、使用场景 |
| synthesis.md | synthesis | 来源对比、交叉验证、综合结论 |
| personal.md | personal | 偏好、规则、历史变化 |

**实现**：
1. `core/templates.py`：`TemplateManager` 管理模板加载和查询
2. MCP 工具：`get_template(facet)` + `list_templates()`
3. CLI：`linglong kb template list` + `linglong kb template get {facet}`

**结果**：8 个模板全部到位，测试 20/20 通过，全量 273/273 通过。

**关联**：`src/linglong/core/templates.py`、`~/linglong/templates/`
