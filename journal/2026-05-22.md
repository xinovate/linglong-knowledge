## 2. facet 分类体系审核

**问题**：94/142 条实体全在 source facet，OpenClaw 同步时映射粒度不够细。

**方案**：
- 旧 7 分面：source / entity / concept / synthesis / experience / methodology / personal
- 新 6 分面：concept / experience / methodology / **project** / **reference** / personal
- 移除：source（拆入 project/reference/concept）、entity（合入 concept）、synthesis（合入 concept）
- 新增：Entity.group 字段（语义子目录）

**执行**：
1. Phase 1：synthesis→concept 2 条确定性迁移
2. Phase 2：LLM（glm-4-flash）重分类 94 条 source → 6 新分面
3. LLM 分配 group：14 个语义子目录（如 openclaw、agent-system、template 等）
4. Entity.group 字段：DB schema + store + MCP + CLI 全链路支持
5. 拥挤检测：lint + MCP + CLI 三层防护

**结果**：
- 142 条全部迁移完成，276 测试通过
- 最终分布：methodology(41) > project(38) > concept(30) > experience(23) > personal(9) > reference(1)

**状态**：✅ 已完成

## 6. output_log 表

**问题**：composer→dispatch 发布后无追踪，可能重复消费同一 entity。

**计划**：knowledge SQLite 新增 `output_log` 表：
```sql
CREATE TABLE output_log (
    id INTEGER PRIMARY KEY,
    entity_id TEXT NOT NULL,
    article_title TEXT,
    publisher TEXT NOT NULL,
    published_at TEXT NOT NULL
);
```
