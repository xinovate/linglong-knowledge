## 写作要求（按优先级排序）

### 1. 核心洞察优先

先问自己：这些记忆中最值得分享的技术认知是什么？
文章应该围绕这个认知展开，而不是围绕记忆本身。

### 2. 标题即观点

- 10-18个汉字
- 标题本身就是一个技术判断或经验结论
- 好例子："MCP协议让工具调用从硬编码变成配置化"、" Draft Mode 是AI内容生产的必要止损机制"
- 坏例子："5月11日学习记录"、"MCP协议学习笔记"

### 3. 摘要：30-40个汉字

- 包含"问题/场景 + 核心结论"
- 好例子："AI内容流水线直接发布导致低质量文章上线，引入 Draft Mode 人工审核层作为止损机制。"
- 坏例子："本文记录了今天学习MCP协议的过程。"

### 4. 正文结构：三段式递进

- 第一段（## 背景/问题）：用一个具体场景引入，说明遇到了什么问题
- 第二段（## 探索/实践）：描述尝试的过程，包含关键代码或配置（如有）
- 第三段（## 结论/洞察）：提炼出可复用的经验，明确说出"应该怎么做"
- 段落之间要有逻辑衔接，不是独立的板块

### 5. 标签与分类

- 标签：3-5个技术关键词（如 "MCP", "AI-Composer", "Content-Quality"）
- 分类：1-2个，如"技术"、"架构"、"AI工程"、"工具链"
